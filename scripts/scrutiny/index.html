The Scrutiny Framework provides an extensible plugin based architecture for building security auditing tools for Fuchsia. Developers can create their own plugins that interact with the system through both the web interface and the command line shell.

Data is collected from the system through custom Data Collectors which populate the Data Model. You can schedule the collectors to run using the Schedule button above. Once these collectors run the controllers then operate on this data to provide a dynamic API surface based on which plugins are loaded.

Data Visualizers are simple web views built on top of the API surface to help improve the usability of the tool. However all commands and REST requests can also be directly passed as command line arguments. 

fx scrutiny -c 'components`

Please refer to the included README.md for more information.
